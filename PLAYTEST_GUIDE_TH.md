# PLAYTEST GUIDE TH

## เป้าหมาย

ทดสอบว่า Animal Score Card Game Prototype v0.3 เล่นแบบ Local Hot-seat ได้จนจบ เข้าใจกติกาได้จริง และเก็บข้อมูลปัญหาเพื่อใช้ตัดสินใจในรอบถัดไปโดยไม่เปลี่ยนสมดุลจากความเห็นเดี่ยว

## กลุ่มทดสอบที่แนะนำ

- ผู้ทดสอบ 5-10 คน
- อย่างน้อย 20 match ที่เล่นจบ
- เล่นบน desktop และ mobile browser เท่าที่ทำได้

## วิธีเล่น Local Hot-seat

1. เปิดเกมใน browser
2. ให้ผู้เล่น 1 และผู้เล่น 2 ใช้อุปกรณ์เดียวกัน
3. เมื่อจบเทิร์น ให้ส่งเครื่องเฉพาะตอนหน้าจอส่งเครื่องแสดงขึ้น
4. ผู้เล่นถัดไปกดพร้อมเล่นด้วยตัวเอง
5. เล่นจนมีผู้ชนะหรือครบ Turn Limit

## Privacy Handoff

- ห้ามส่งเครื่องระหว่างที่มือของผู้เล่นยังแสดงอยู่
- ใช้หน้าจอส่งเครื่องทุกครั้งหลังจบเทิร์น
- ผู้เล่นคนถัดไปควรกดปุ่มพร้อมเล่นเอง

## Resume Saved Game

- เกมที่ยังไม่จบถูกบันทึกใน Local Storage ของ browser
- กลับมาที่ browser เดิมและกดเล่นต่อจากเซฟเดิม
- เซฟผูกกับ browser และอุปกรณ์นั้นเท่านั้น

## Export Match Log

1. เปิดเมนูหรือหน้าผลการแข่งขัน
2. กดส่งออกไฟล์เซฟ
3. ถ้า Clipboard API ใช้ไม่ได้ ให้คัดลอก JSON จากหน้าต่างส่งออก
4. แนบ JSON ใน issue เมื่อรายงานบั๊กถ้ามี

## Export Playtest Feedback

1. เล่น match ให้จบ
2. ที่หน้าผลการแข่งขัน กดฟีดแบ็ก Human Playtest (ไม่บังคับ)
3. เลือกบทบาทผู้ให้ฟีดแบ็ก: P1, P2, BOTH หรือ OBSERVER
4. กรอก rating 1-5 ให้ครบทุกช่อง: ความชัดเจนของกติกา, ความสนุก, ความยาวเกม, สมดุลเกม, ความชัดเจนของ UI
5. กรอกรหัสผู้ทดสอบนิรนามได้ถ้าต้องการ ห้ามใส่ชื่อจริง อีเมล เบอร์โทร wallet address หรือข้อมูลส่วนตัว
6. กรอกข้อความเสริม เช่น จุดที่สับสน การ์ดที่รู้สึกว่าแข็ง/อ่อน รายละเอียดบั๊ก หรือความคิดเห็นเพิ่มเติม
7. กดบันทึกและส่งออก JSON ระบบจะเก็บฟีดแบ็กไว้ใน browser และดาวน์โหลดไฟล์ `human-feedback-<matchId>-<timestamp>.json`

## Export Match History

1. จากเมนูหลัก กดประวัติการเล่น
2. กดส่งออกประวัติทั้งหมด เพื่อดาวน์โหลดไฟล์ `animal-score-match-history-<timestamp>.json`
3. ในแต่ละรายการ กดส่งออก match นี้ เพื่อดาวน์โหลดไฟล์ `animal-score-match-<matchId>.json`
4. ถ้าไม่มีประวัติ ระบบจะแสดงว่าไม่มีประวัติการเล่นและไม่ลบข้อมูลใด
5. ปุ่มลบประวัติทั้งหมดยังอยู่ที่เดิม และต้องยืนยันก่อนลบเสมอ

โดยปกติ browser จะเก็บไฟล์ที่ดาวน์โหลดไว้ในโฟลเดอร์ Downloads ของเครื่อง หรือโฟลเดอร์ที่ผู้ใช้ตั้งค่าไว้ใน browser

## สิ่งที่ควรบันทึก

- ผู้เล่นเข้าใจกติกาเร็วแค่ไหน
- จุดที่ผู้เล่นลังเลหรือใช้ผิด
- เวลาต่อ match
- การ์ดที่ถูกมองว่าแข็งหรืออ่อนซ้ำหลายครั้ง
- ปัญหา UI, ขนาดหน้าจอ, keyboard, หรือการส่งเครื่อง
- บั๊กที่ทำให้เกมค้างหรือผลลัพธ์ผิด

## ระดับความรุนแรงของบั๊ก

- Blocker: เล่นต่อหรือจบเกมไม่ได้
- Critical: กติกาหรือคะแนนผิดจนผล match ไม่น่าเชื่อถือ
- Major: ฟีเจอร์สำคัญใช้ยากหรือผิด แต่ยังมีทางเล่นต่อ
- Minor: ข้อความ, layout, หรือความไม่สะดวกเล็กน้อย

## คำเตือนเรื่อง Balance

อย่าเปลี่ยน balance จากความเห็นเดี่ยวหรือ match เดียว ให้มองหาข้อมูลซ้ำจากหลาย match หลายคู่ผู้เล่น และดูทั้ง match log, feedback, และพฤติกรรมจริงก่อนเสนอ patch

## Privacy และชนิดของข้อมูลส่งออก

- Match history summary คือประวัติย่อที่บันทึกไว้หลังจบ match เช่น ผู้ชนะ คะแนน จำนวนเทิร์น ระยะเวลา และสถิติสรุป ไม่มี action log เต็ม
- Full match log คือไฟล์เซฟ/log ที่ส่งออกจากหน้าผลการแข่งขัน มี state และ action log สำหรับ debug หรือ resume
- Human feedback คือแบบฟอร์มความคิดเห็นของผู้เล่นหรือผู้สังเกตการณ์ ผูกกับ matchId แต่ไม่เปลี่ยน match state หรือประวัติ match
- ข้อมูลทั้งหมดอยู่ใน browser localStorage และดาวน์โหลดในเครื่องเท่านั้น ไม่มี backend, accounts, analytics, cookies หรือ network request
- ห้ามเก็บชื่อจริง อีเมล เบอร์โทร wallet address, IP address หรือข้อมูลส่วนตัวอื่นในช่องฟีดแบ็ก
