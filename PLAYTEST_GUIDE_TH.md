# PLAYTEST GUIDE TH

## เป้าหมาย

ทดสอบว่า Animal Score Card Game Prototype v0.3 เล่นแบบ Local Hot-seat ได้จนจบ เข้าใจกติกาได้จริง และเก็บข้อมูลปัญหาเพื่อใช้ตัดสินใจในรอบถัดไปโดยไม่เปลี่ยนสมดุลจากความเห็นเดี่ยว

## กลุ่มทดสอบที่แนะนำ

- ผู้ทดสอบ 5-10 คน
- อย่างน้อย 20 match ที่เล่นจบ
- เล่นบน desktop และ mobile browser เท่าที่ทำได้

## วิธีเล่น Local Hot-seat

1. เปิดเกมใน browser
2. ให้ผู้เล่น 1 และผู้เล่น 2 ใช้อุปกรณ์เดียวกัน
3. เมื่อจบเทิร์น ให้ส่งเครื่องเฉพาะตอนหน้าจอส่งเครื่องแสดงขึ้น
4. ผู้เล่นถัดไปกดพร้อมเล่นด้วยตัวเอง
5. เล่นจนมีผู้ชนะหรือครบ Turn Limit

## Privacy Handoff

- ห้ามส่งเครื่องระหว่างที่มือของผู้เล่นยังแสดงอยู่
- ใช้หน้าจอส่งเครื่องทุกครั้งหลังจบเทิร์น
- ผู้เล่นคนถัดไปควรกดปุ่มพร้อมเล่นเอง

## Resume Saved Game

- เกมที่ยังไม่จบถูกบันทึกใน Local Storage ของ browser
- กลับมาที่ browser เดิมและกดเล่นต่อจากเซฟเดิม
- เซฟผูกกับ browser และอุปกรณ์นั้นเท่านั้น

## Export Match Log

1. เปิดเมนูหรือหน้าผลการแข่งขัน
2. กดส่งออกไฟล์เซฟ
3. ถ้า Clipboard API ใช้ไม่ได้ ให้คัดลอก JSON จากหน้าต่างส่งออก
4. แนบ JSON ใน issue เมื่อรายงานบั๊กถ้ามี

## Export Playtest Feedback

1. เล่น match ให้จบ
2. ที่หน้าผลการแข่งขัน กดส่งออกฟีดแบ็ก Playtest
3. กรอก rating 1-5 เฉพาะช่องที่ต้องการ
4. กรอกข้อความเสริมถ้ามี
5. กดส่งออก JSON และเก็บไฟล์หรือคัดลอกข้อความ

## สิ่งที่ควรบันทึก

- ผู้เล่นเข้าใจกติกาเร็วแค่ไหน
- จุดที่ผู้เล่นลังเลหรือใช้ผิด
- เวลาต่อ match
- การ์ดที่ถูกมองว่าแข็งหรืออ่อนซ้ำหลายครั้ง
- ปัญหา UI, ขนาดหน้าจอ, keyboard, หรือการส่งเครื่อง
- บั๊กที่ทำให้เกมค้างหรือผลลัพธ์ผิด

## ระดับความรุนแรงของบั๊ก

- Blocker: เล่นต่อหรือจบเกมไม่ได้
- Critical: กติกาหรือคะแนนผิดจนผล match ไม่น่าเชื่อถือ
- Major: ฟีเจอร์สำคัญใช้ยากหรือผิด แต่ยังมีทางเล่นต่อ
- Minor: ข้อความ, layout, หรือความไม่สะดวกเล็กน้อย

## GitHub Issue Labels ที่แนะนำ

## Acceptance Criteria สำหรับ Phase 7

- เปิดเกมผ่าน build production ได้
- GitHub Pages workflow พร้อมใช้งาน
- เล่น Local Hot-seat จบ match ได้
- Export match log และ playtest feedback JSON ได้
- Issue templates พร้อมรับ bug, playtest, และ balance feedback
- เอกสาร playtest ระบุขั้นตอนและข้อจำกัดชัดเจน

## คำเตือนเรื่อง Balance

อย่าเปลี่ยน balance จากความเห็นเดี่ยวหรือ match เดียว ให้มองหาข้อมูลซ้ำจากหลาย match หลายคู่ผู้เล่น และดูทั้ง match log, feedback, และพฤติกรรมจริงก่อนเสนอ patch
